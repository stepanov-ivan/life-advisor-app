# Release checklist

This checklist is intended for the first public release and future lightweight releases.

## Repository presentation

- README reflects current product behavior
- screenshots or demo are up to date
- roadmap and changelog are updated
- contribution and security docs still match reality

## Product quality

- tests pass locally
- CI passes on default branch
- onboarding flow is working
- settings and LLM setup flow are understandable

## Public review readiness

- no committed secrets or local artifacts
- release notes summarize user-visible changes
- at least a few contributor-friendly issues exist
- repository About text and topics match the project direction

## Suggested first release title

- `v0.1.0 - Public OSS baseline`

## Suggested release notes structure

- what the app does
- what changed since the public repository cleanup
- known limitations
- where contributors can help next
