## ADDED Requirements

### Requirement: Execution mode на home surface meal domain
Система SHALL выполнять meal-цепочку на home surface meal domain, которым является дашборд, без открытия отдельного flow-экрана.

#### Scenario: Запуск meal execution из другой вкладки
- **WHEN** пользователь запускает meal-сценарий, находясь не на вкладке дашборда
- **THEN** система переключает его на дашборд и активирует execution mode там

#### Scenario: Активная карточка шага
- **WHEN** execution mode перешёл к meal step
- **THEN** соответствующая карточка приёма пищи на дашборде становится активным контекстом шага

### Requirement: Прогресс цепочки и отмена хвоста
Система SHALL показывать progress bar шагов с группировкой по дням и позволять отменить неподтверждённый хвост цепочки через явный exit flow.

#### Scenario: Прогресс по нескольким дням
- **WHEN** execution chain содержит шаги за несколько дней
- **THEN** progress UI показывает шаги сгруппированными по дням

#### Scenario: Отмена незавершённой цепочки
- **WHEN** пользователь нажимает кнопку закрытия execution mode при наличии неподтверждённых шагов
- **THEN** система предупреждает о потере хвоста цепочки и после подтверждения удаляет все неподтверждённые шаги

### Requirement: Локальное редактирование meal draft
Система SHALL позволять пользователю внутри execution mode локально править только состав и порции meal draft-а.

#### Scenario: Изменение только продуктов и порций
- **WHEN** пользователь редактирует активный meal draft
- **THEN** доступны только действия `add product`, `remove product` и `change portion`

#### Scenario: Date и meal slot недоступны для прямой правки
- **WHEN** пользователь находится на активном шаге execution mode
- **THEN** `date` и `mealSlot` отображаются как read-only аргументы шага

### Requirement: Автопереход к следующему шагу после commit
Система SHALL после успешного подтверждения шага автоматически переходить к следующему шагу цепочки.

#### Scenario: Успешный commit текущего шага
- **WHEN** пользователь подтверждает текущий meal draft и сохранение прошло успешно
- **THEN** система коммитит шаг, показывает краткое подтверждение и автоматически активирует следующий шаг

#### Scenario: Последний шаг цепочки
- **WHEN** пользователь подтверждает последний шаг execution chain
- **THEN** execution mode завершается в состоянии `completed`
