## ADDED Requirements

### Requirement: Классификация meal-сценария
Система SHALL классифицировать meal-сообщение пользователя как `meal-create` или `meal-edit` до построения шагов выполнения.

#### Scenario: Сообщение о создании приёмов пищи
- **WHEN** пользователь отправляет сообщение с новым описанием еды без явной цели редактирования
- **THEN** система классифицирует его как `meal-create`

#### Scenario: Сообщение о правке существующего приёма
- **WHEN** пользователь отправляет сообщение с намерением изменить уже существующую запись о еде
- **THEN** система классифицирует его как `meal-edit`

### Requirement: Определение временного контекста
Система SHALL определять релевантный временной контекст meal-сообщения до этапа планирования.

#### Scenario: Явно указан относительный день
- **WHEN** пользователь пишет запрос со словами `сегодня`, `вчера` или `завтра`
- **THEN** система использует этот день как явный day scope для дальнейшего planning

#### Scenario: День не указан явно
- **WHEN** пользователь отправляет meal-сообщение без прямого указания дня
- **THEN** система определяет day scope из доступного контекста перед построением шагов
