## ADDED Requirements

### Requirement: Построение цепочки нормализованных шагов
Система SHALL преобразовывать meal-сообщение в цепочку нормализованных шагов `create(date, mealSlot, productsDraft)` или `edit(date, mealSlot, productsDraft)`.

#### Scenario: Один шаг для одного приёма пищи
- **WHEN** сообщение пользователя описывает один конкретный приём пищи
- **THEN** planner возвращает один нормализованный шаг соответствующего типа

#### Scenario: Несколько шагов для одного сообщения
- **WHEN** сообщение пользователя обоснованно относится к нескольким приёмам пищи
- **THEN** planner возвращает цепочку из нескольких нормализованных шагов

### Requirement: Planner создаёт только обоснованные шаги
Система SHALL создавать только те meal steps, которые можно обосновать по сообщению пользователя и доступному контексту.

#### Scenario: Неполный день не форсируется в полный шаблон
- **WHEN** из сообщения удалось надёжно вывести только два приёма пищи
- **THEN** planner возвращает только эти два шага и не добавляет лишний meal slot

#### Scenario: Отсутствие достаточных оснований для шага
- **WHEN** planner не может обосновать дополнительный meal step
- **THEN** этот шаг не включается в итоговую цепочку

### Requirement: Детерминированный порядок multi-day шагов
Система SHALL группировать шаги по дням и упорядочивать их детерминированно внутри каждой day group.

#### Scenario: Шаги за несколько дней
- **WHEN** planner вернул шаги для нескольких дней
- **THEN** итоговая цепочка группируется по дням, а внутри дня шаги идут по порядку meal slots

#### Scenario: Переход к следующему дню
- **WHEN** шаги текущего дня завершены
- **THEN** следующий активный шаг выбирается из следующей day group

### Requirement: V1 не поддерживает смешанные create/edit цепочки
Система SHALL в первой версии строить только чистые цепочки `meal-create` или `meal-edit`, но не смешивать их в одной agent-сессии.

#### Scenario: Чистая create-цепочка
- **WHEN** сценарий классифицирован как `meal-create`
- **THEN** все шаги итогового плана имеют тип `create`

#### Scenario: Чистая edit-цепочка
- **WHEN** сценарий классифицирован как `meal-edit`
- **THEN** все шаги итогового плана имеют тип `edit`
