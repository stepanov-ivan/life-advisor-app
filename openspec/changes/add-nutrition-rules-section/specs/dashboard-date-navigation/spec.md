## ADDED Requirements

### Requirement: Четвёртый таб «Правила»
Система SHALL добавлять четвёртый таб «Правила» в `TabView` приложения рядом с Дашборд, Аналитика, Настройки.

#### Scenario: Открытие вкладки Правила
- **WHEN** пользователь тапает по табу «Правила»
- **THEN** система показывает список правил питания с индикаторами зон

#### Scenario: Переключение между табами
- **WHEN** пользователь переключается с Дашборда на Правила и обратно
- **THEN** состояние каждого таба сохраняется независимо

### Requirement: Навигация от нарушения к приёму
Система SHALL при тапе по нарушению на экране детали правила переключать таб на Дашборд, устанавливать выбранную дату и открывать редактор приёма.

#### Scenario: Переход к приёму-нарушителю
- **WHEN** пользователь тапает по нарушению с привязанным `MealEvent`
- **THEN** система: записывает дату нарушения в `selectedDate` на `ContentView`, переключает `TabView` на Дашборд, открывает `MealEventEditorView` для этого приёма

#### Scenario: Нарушение без привязанного приёма
- **WHEN** нарушение не привязано к конкретному `MealEvent`
- **THEN** система переключает таб на Дашборд и устанавливает дату, но не открывает редактор

## MODIFIED Requirements

### Requirement: Навигация по дням на дашборде
Система SHALL предоставлять навигацию по дням через компактный week-strip и выбор даты в календаре для просмотра логов выбранного дня. `selectedDate` хранится на уровне `ContentView` и пробрасывается как `@Binding` в `DashboardView`.

#### Scenario: Переключение дня через week-strip
- **WHEN** пользователь выбирает день в week-strip
- **THEN** дашборд показывает данные приёмов пищи за выбранную дату

#### Scenario: Выбор даты через календарь
- **WHEN** пользователь выбирает дату в календаре
- **THEN** дашборд переключается на выбранную дату, а week-strip перестраивается на календарную неделю этой даты

#### Scenario: Установка даты извне
- **WHEN** другой экран (например, деталь правила) устанавливает `selectedDate`
- **THEN** дашборд переключается на эту дату и week-strip обновляется

### Requirement: Ограничение будущих дат
Система SHALL запрещать выбор дат позже текущего дня в week-strip и календаре.

#### Scenario: Недоступная будущая дата
- **WHEN** пользователь пытается выбрать будущую дату
- **THEN** дата не применяется, а система показывает подсказку «Будущие даты недоступны»

### Requirement: Восстановление последней выбранной даты
Система SHALL сохранять и восстанавливать последнюю выбранную пользователем дату дашборда между сессиями.

#### Scenario: Повторное открытие дашборда
- **WHEN** пользователь повторно открывает дашборд
- **THEN** система восстанавливает последнюю выбранную дату, если она не позже текущего дня

#### Scenario: Устаревшая сохранённая дата в будущем
- **WHEN** сохранённая дата оказывается позже текущего дня
- **THEN** система автоматически выбирает текущий день
