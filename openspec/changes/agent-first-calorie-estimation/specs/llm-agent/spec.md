## MODIFIED Requirements

### Requirement: Структуризация текста через LLM
Система SHALL отправлять текст логирования в LLM и получать строгий JSON-контракт оценки питания, включающий режим оценки, итоговые калории/БЖУ, confidence и explainability.

#### Scenario: Успешная агентская оценка
- **WHEN** система отправляет текст «овсянка с ягодами и кофе с молоком»
- **THEN** LLM возвращает валидный JSON с `mode`, `totals`, `confidence`, `items[]`, `impact_score`, `reason`, `modelId`, `promptVersion`, `estimationSchemaVersion`

#### Scenario: Ответ нарушает схему
- **WHEN** LLM возвращает JSON без обязательных полей или с неконсистентными данными
- **THEN** система помечает событие как `parse_failed` без автоматического retry

### Requirement: Рекомендация на оставшиеся приёмы
Система SHALL предоставлять пользователю возможность запросить LLM-рекомендацию по распределению калорий и БЖУ на оставшиеся незаполненные окна питания.

#### Scenario: Запрос рекомендации
- **WHEN** пользователь залогировал завтрак (structured), открыл вкладку «Рекомендация» на слоте «Обед» и нажал «Получить рекомендацию»
- **THEN** система отправляет в LLM: съеденные приёмы с ингредиентами и БЖУ, цель, оставшиеся окна → LLM возвращает распределение калорий и БЖУ на оставшиеся приёмы с текстовыми рекомендациями

#### Scenario: Рекомендация без залогированных приёмов
- **WHEN** пользователь открывает вкладку «Рекомендация», но ни один приём ещё не залогирован
- **THEN** кнопка «Получить рекомендацию» неактивна с подсказкой «Заполните хотя бы один приём пищи»

#### Scenario: Кеширование рекомендации
- **WHEN** пользователь получает рекомендацию, а затем открывает вкладку «Рекомендация» на другом слоте
- **THEN** отображается последняя сохранённая рекомендация за сегодня, без повторного LLM-запроса. При новом логировании кеш сбрасывается

### Requirement: Отображение прогресса LLM-запроса
Система SHALL показывать прогресс-бар и данные, которые будут отправлены в LLM, перед выполнением запроса.

#### Scenario: Прогресс при оценке логирования
- **WHEN** запущена обработка нового логирования
- **THEN** отображается прогресс-статус и краткая сводка отправляемого текста

#### Scenario: Прогресс при совете дня
- **WHEN** пользователь нажимает «Совет дня»
- **THEN** отображается прогресс-бар с текстом «Анализирую ваш день...», сводка отправляемых данных (калории, БЖУ, список блюд)

## ADDED Requirements

### Requirement: Валидация и нормализация explainability-полей
Система SHALL валидировать `impact_score` и explainability-поля ответа LLM перед сохранением.

#### Scenario: Impact score вне диапазона
- **WHEN** LLM возвращает `impact_score` меньше 0 или больше 1
- **THEN** система принудительно ограничивает значение в диапазоне [0..1] перед сохранением

### Requirement: Версионирование оценки и хранение raw ответа
Система SHALL сохранять метаданные версии оценки и raw JSON ответа LLM для дебага и трассировки.

#### Scenario: Сохранение версии модели
- **WHEN** получен валидный ответ LLM
- **THEN** система сохраняет `modelId`, `promptVersion`, `estimationSchemaVersion` и raw payload в локальное хранилище
