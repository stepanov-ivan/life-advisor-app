## Why

Текущий интерфейс логирования и просмотра питания перегружен техническими деталями и не поддерживает базовый сценарий просмотра истории по дням. Это снижает читаемость дашборда, затрудняет ручную правку структуры блюда и делает часть советов непонятной для пользователя.

## What Changes

- Добавить на дашборд выбор даты (календарь) с возможностью переключаться между днями и просматривать логи за прошлые/будущие даты.
- Упростить карточку приёма пищи на дашборде: убрать отображение технических меток (`Уверенность`, `Из памяти`) из основного списка.
- Исправить синхронизацию текста и структуры в редакторе: при удалении позиции из текстового описания корректно удалять/обновлять соответствующую структурную позицию.
- Переименовать секцию `Ручная корректировка по позициям` в более понятное пользовательское название (например, `Состав блюда`).
- Добавить явные заголовки столбцов в блоке структуры (`Ккал`, `Б`, `Ж`, `У`) для улучшения читаемости.
- Упростить explainability-блок:
- Переименовать `Почему этот совет` в `Совет`.
- Убрать технические поля (`источник структуры`, `уверенность модели`, `режим оценки`) из пользовательского текста совета.
- Для логов из памяти показывать только полезный совет/рекомендацию без служебной информации.

## Capabilities

### New Capabilities
- `dashboard-date-navigation`: навигация по календарным дням на дашборде и просмотр логов выбранной даты.

### Modified Capabilities
- `event-logging`: обновление UX дашборда и редактора структуры (синхронизация текста/позиций, названия секций, заголовки столбцов).
- `insight-engine`: упрощение формата совета и удаление технической explainability-информации из пользовательского UI.

## Impact

- Затронутые экраны: `DashboardView`, редактор события приёма пищи, блок совета.
- Затронутые модели/поток: фильтрация `MealEvent` по выбранной дате, логика синхронизации текста и структуры.
- Возможные изменения тестов: unit/integration тесты на фильтрацию по дате, отображение упрощённого совета, и поведение удаления позиции из структуры.
