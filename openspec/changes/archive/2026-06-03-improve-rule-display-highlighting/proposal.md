## Why

Правила питания на дашборде сейчас показывают только цветной индикатор (красный/зелёный/жёлтый) без числовых значений — пользователь не видит, насколько он близок к порогу. Нарушения в карточках приёмов пищи отображаются с английскими кодами причин (`exceeds_upper`, `below_lower`), которые непонятны русскоязычному пользователю. Отсутствует третий уровень подсветки — на уровне продукта внутри карточки приёма пищи, хотя данные о вкладе продукта в нарушение уже вычисляются и хранятся в `contributionJSON`.

## What Changes

- **Числовые значения на дашборде**: `DayAnalyticsPanel` вместо «В норме» / «Нарушено» показывает фактическое значение и порог правила (например, «14.2% из ≤10%» для сахара, «45 г из ≥30 г» для клетчатки)
- **Русские причины нарушений**: `reasonCode` заменяется на человекочитаемое описание на русском языке (например, `exceeds_upper` → «Превышение: 14.2% при норме ≤10%»)
- **Подсветка продукта в карточке приёма**: ингредиенты с вкладом >30% в нарушенное правило подсвечиваются в карточке `MealSlotCard` с указанием процента вклада и названия правила
- **Цветовая схема сохраняется**: красный (violation), жёлтый (warning), зелёный (normal), серый (no data) — но теперь с числовым контекстом

## Capabilities

### New Capabilities

- `rule-value-display`: отображение расчётных значений метрик правил вместе с порогами в аналитической панели дашборда вместо одних только цветовых индикаторов
- `russian-violation-reasons`: человекочитаемые описания причин нарушений на русском языке, заменяющие технические английские коды
- `product-violation-highlighting`: подсветка конкретных продуктов в карточке приёма пищи, чей вклад в нарушенное правило превышает 30%

### Modified Capabilities

- `day-analytics-panel`: панель дашборда теперь показывает числовые значения метрик
- `meal-rule-feedback`: карточка приёма пищи теперь показывает русские описания нарушений и подсветку продуктов

## Impact

- `DayAnalyticsPanel.swift` — переработка строки правила для показа значений
- `MealSlotCard.swift` — замена `reasonCode` на русский текст, добавление подсветки продуктов
- `RuleEngine.swift` — добавление метода для генерации русских описаний нарушений
- `RuleEvaluation.swift` — возможное добавление форматированных сообщений в `EvaluationResult`
- `nutrition_rules.json` — может потребоваться поле `unit` (единицы измерения) для форматирования значений
- `RuleViolation.swift` — без изменений (модель уже содержит все нужные данные)
