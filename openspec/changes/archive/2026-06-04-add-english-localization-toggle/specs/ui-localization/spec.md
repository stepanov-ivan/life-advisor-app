## ADDED Requirements

### Requirement: Единый язык приложения с пользовательским override
Система SHALL определять effective language приложения на основе системной локали по умолчанию и пользовательского override `System | English | Русский`, сохраняемого между запусками.

#### Scenario: Использование системного языка по умолчанию
- **WHEN** пользователь не выбирал override языка и системная локаль устройства английская
- **THEN** приложение запускается на английском языке

#### Scenario: Переопределение системного языка пользователем
- **WHEN** системная локаль устройства русская, а пользователь выбирает `English` в настройках
- **THEN** effective language приложения становится английским и не зависит от системной локали до изменения настройки

### Requirement: Смена языка применяется без перезапуска
Система SHALL применять изменение языка приложения ко всем основным пользовательским поверхностям без перезапуска приложения.

#### Scenario: Переключение языка в настройках
- **WHEN** пользователь меняет язык приложения с `Русский` на `English` в настройках
- **THEN** экран настроек и другие основные экраны начинают отображать локализованный текст на английском без перезапуска приложения

### Requirement: Английский является базовым fallback-языком
Система SHALL использовать английский как fallback-язык для отсутствующих локализационных ключей во всех поддерживаемых таблицах локализации.

#### Scenario: Отсутствует русский перевод ключа
- **WHEN** effective language приложения русский, но для ключа отсутствует русская локализация
- **THEN** пользователь видит английское значение этого ключа

### Requirement: Локализация разделена по доменным таблицам
Система SHALL хранить локализуемые строки как минимум в отдельных таблицах `Localizable`, `Rules` и `Prompts`.

#### Scenario: Обычный UI-текст
- **WHEN** приложение отображает кнопку, placeholder или заголовок обычного экрана
- **THEN** строка берётся из таблицы `Localizable`

#### Scenario: Текст правила питания
- **WHEN** приложение отображает название правила, категорию правила или explanation fragment нарушения
- **THEN** строка берётся из таблицы `Rules`

#### Scenario: Prompt template
- **WHEN** приложение собирает prompt для LLM
- **THEN** локализуемые prompt fragments берутся из таблицы `Prompts`

### Requirement: Полнота локализаций проверяется автоматически
Система SHALL иметь автоматическую проверку, которая валит тесты, если ключ отсутствует хотя бы в одной из официально поддерживаемых локализаций `ru` или `en` для поддерживаемых localization tables.

#### Scenario: Отсутствует английский перевод
- **WHEN** в таблице `Rules` присутствует ключ только для `ru`, но отсутствует для `en`
- **THEN** automated test падает с ошибкой о неполной локализации
