## Why

Текущее поведение правил питания не даёт пользователю понятного объяснения, какие именно продукты сформировали значение правила за день и почему правило считается нарушенным или нормальным. Отдельный экран аналитики правила добавляет лишнюю навигацию, а текущая подсветка на уровне приёма пищи и продукта опирается на хрупкую модель вклада и работает непоследовательно.

## What Changes

- Убрать переход из правила в отдельный экран аналитики и заменить его inline-раскрытием правила прямо в `DayAnalyticsPanel`.
- Удалить пользовательский сценарий с экраном детали правила и недельным графиком из текущего UX.
- Показывать в `DayAnalyticsPanel` только дневные правила, для которых можно честно вычислить вклад продуктов за выбранный день.
- Для каждого такого правила показывать текущее значение метрики, а при раскрытии — короткий summary, человекочитаемую норму и список продуктов-вкладчиков.
- Показывать в раскрытии top-3 продукта по вкладу с возможностью раскрыть оставшийся хвост и свернуть его обратно.
- Использовать единый формат представления вклада: для граммовых и категориальных правил — абсолютное значение с долей в скобках, для процентных правил — только абсолютное значение метрики.
- Упростить модель оценки правил: убрать `warning` как продуктовую зону и оставить бинарную модель `normal` / `violation` (плюс `no_data`).
- Убрать подсветку на уровне карточки приёма как отдельный summary-слой и оставить только подсветку конкретных продуктов, если их вклад в нарушенное правило превышает 20%.
- Оставить раскрытие продукта в `MealSlotCard`, но использовать его только для списка нарушенных правил, в которые продукт внёс значимый вклад.
- Удалить `contributionJSON` из модели нарушений и заменить его новой persist-моделью дневного explainability: `RuleContributionSnapshot` и `RuleContributionItem`.
- Перевести rule tracing, breakdown и meal-level highlighting на новую модель связей вместо парсинга JSON.

## Capabilities

### New Capabilities
- `rule-contribution-breakdown`: inline-объяснение дневного правила через summary метрики, норму и список продуктов-вкладчиков.
- `rule-contribution-snapshots`: сохранение explainability-снимка правила за день и связанных вкладов продуктов как отдельной локальной модели данных.

### Modified Capabilities
- `meal-rule-feedback`: меняется роль карточки приёма пищи; убирается summary нарушений на уровне приёма, остаётся только product-level сигнал и список затронутых правил под конкретным продуктом.
- `product-violation-highlighting`: меняются критерии и источник данных для подсветки продукта; подсветка строится по persisted contribution items и бинарной модели нарушения.
- `rule-violation-tracing`: tracing больше не использует `contributionJSON`; связь правила, продукта и приёма строится через новые persisted contribution entities.
- `rule-analytics`: меняется interaction model списка правил; tap по правилу раскрывает дневной breakdown продуктов вместо перехода в отдельную аналитику.
- `nutrition-rules`: меняется продуктовая модель оценки правил; warning-зона и approaching-сценарии исключаются из пользовательского поведения.

## Impact

- Затронутые экраны: `DayAnalyticsPanel`, `MealSlotCard`, экран/список правил, текущий экран детали правила.
- Затронутые сервисы и модели: `RuleEngine`, rule evaluation, rule tracing, contribution calculation, `RuleViolation`, новые snapshot/item сущности explainability.
- Затронуто локальное хранилище: потребуется новая persisted модель дневных contribution snapshots и удаление зависимости от `contributionJSON`.
- Потребуются обновления unit/UI тестов для бинарной модели правил, inline breakdown правил и product-level highlighting.
