## MODIFIED Requirements

### Requirement: Аналитическая панель на дашборде
Система SHALL показывать в `DayAnalyticsPanel` только дневные правила, для которых можно честно вычислить product contribution breakdown за выбранный день. Правила, требующие недельного анализа, SHALL NOT отображаться в этой панели.

#### Scenario: Дневные правила отображаются
- **WHEN** пользователь открывает дашборд и за выбранный день есть structured-приёмы
- **THEN** `DayAnalyticsPanel` показывает только поддерживаемые дневные правила и их текущее состояние

#### Scenario: Недельные правила скрыты
- **WHEN** правило требует недельного анализа или weekly window
- **THEN** оно не показывается в `DayAnalyticsPanel`

#### Scenario: Дашборд без данных за день
- **WHEN** за выбранный день нет данных для дневных правил
- **THEN** `DayAnalyticsPanel` показывает пустое состояние без breakdown-строк

### Requirement: Строка правила показывает текущее значение и раскрытие breakdown
Система SHALL показывать в строке правила его название, текущее значение метрики и disclosure indicator только для правил с доступным breakdown. При тапе правило SHALL раскрываться inline, а не переходить на отдельный экран.

#### Scenario: Раскрываемое правило показывает chevron
- **WHEN** у дневного правила есть поддерживаемый breakdown
- **THEN** строка правила показывает disclosure indicator и реагирует на тап

#### Scenario: Нераскрываемое правило без chevron
- **WHEN** у правила нет поддерживаемого breakdown
- **THEN** строка не показывает disclosure indicator и не раскрывается

#### Scenario: Тап по правилу
- **WHEN** пользователь тапает по раскрываемой строке правила
- **THEN** вместо перехода на новый экран под строкой открывается inline breakdown

## REMOVED Requirements

### Requirement: График правила за неделю
**Reason**: Недельный график и отдельный экран правила убраны из пользовательского сценария в пользу inline explainability для дневных правил.
**Migration**: Пользовательский tap по правилу теперь раскрывает breakdown продуктов в `DayAnalyticsPanel`; недельный анализ проектируется отдельно и не является частью этой панели.

### Requirement: Список нарушений на детали правила
**Reason**: Отдельный экран детали правила исключён из текущего UX и заменён inline breakdown без отдельной навигации.
**Migration**: Источники нарушения объясняются внутри `DayAnalyticsPanel` через summary и продукты-вкладчики.

### Requirement: Зоны на графике и в списке
**Reason**: График и warning-based presentation исключены из нового UX; панель использует только `normal`, `violation` и `no_data`.
**Migration**: Все UI surface, связанные с правилами, переходят на бинарную модель зон и не используют жёлтую warning-семантику.
