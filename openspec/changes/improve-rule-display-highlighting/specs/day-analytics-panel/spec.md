## MODIFIED Requirements

### Requirement: Аналитическая панель дня на дашборде
Система SHALL показывать на дашборде под списком приёмов пищи всегда видимую (без `DisclosureGroup`) статическую аналитическую панель с оценкой всех включённых правил за выбранный день, сгруппированных по трём категориям. Для каждого правила SHALL отображаться числовое значение метрики и порог (для range и countSkipped) или зоновый текст (для presence). Расчёт всегда выполняется по текущим данным дня — любое количество приёмов (1, 2, все).

#### Scenario: Панель при наличии данных за день
- **WHEN** за выбранный день есть ≥1 structured-приём и включено 16 правил
- **THEN** панель показывает три группы: «Баланс макронутриентов» (5 правил), «Ограничения» (5 правил), «Качество продуктов и режим» (6 правил) с числовым значением и цветовым индикатором зоны для каждого

#### Scenario: Панель при 2 приёмах из 4 окон
- **WHEN** пользователь ввёл 2 приёма пищи (завтрак и обед), ужин и перекус пропущены
- **THEN** панель оценивает все 16 правил по доступным данным из 2 приёмов

#### Scenario: Панель при отсутствии данных за день
- **WHEN** за выбранный день нет ни одного structured-приёма
- **THEN** панель показывает сообщение «Нет данных за сегодня» и не отображает правила

#### Scenario: Тап по правилу в панели
- **WHEN** пользователь тапает по правилу в аналитической панели
- **THEN** открывается `RuleDetailView` с графиком и списком нарушений за неделю

### Requirement: Цветовая индикация зон в панели
Система SHALL показывать зону каждого правила цветным кругом и цветным числовым значением: зелёный (normal), жёлтый (warning), красный (violation), серый (нет данных).

#### Scenario: Все правила в норме
- **WHEN** все 16 правил показывают зону `normal`
- **THEN** все индикаторы зелёные, все числа зелёные

#### Scenario: Смешанные зоны
- **WHEN** 3 правила нарушены, 2 в warning
- **THEN** 3 индикатора красные, 2 жёлтые, остальные зелёные; числа окрашены соответственно

## ADDED Requirements

### Requirement: Сводная строка над группами правил
Система SHALL показывать над группами правил сводную строку «N из M правил соблюдаются», где N — количество правил в зоне `normal`, M — общее количество включённых правил. Цвет строки SHALL соответствовать худшей зоне среди всех правил.

#### Scenario: Все правила в норме
- **WHEN** все 16 правил включены и показывают `normal`
- **THEN** сводная строка показывает «16 из 16 правил соблюдаются» зелёным цветом

#### Scenario: Часть правил нарушена
- **WHEN** 12 правил `normal`, 3 `warning`, 1 `violation`
- **THEN** сводная строка показывает «12 из 16 правил соблюдаются» красным цветом (худшая зона — violation)

#### Scenario: Одно правило в warning
- **WHEN** 15 правил `normal`, 1 `warning`
- **THEN** сводная строка показывает «15 из 16 правил соблюдаются» жёлтым цветом
