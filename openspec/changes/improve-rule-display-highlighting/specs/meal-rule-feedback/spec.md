## MODIFIED Requirements

### Requirement: Инлайн-список нарушенных правил в карточке приёма
Система SHALL показывать в карточке `MealSlotCard` список правил, нарушенных сегодня, с указанием названия правила и вклада приёма в нарушение. Описание нарушения SHALL генерироваться методом `RuleEngine.violationDescription(for:)` на русском языке. Максимум 3 нарушения в карточке.

#### Scenario: Приём с одним нарушением
- **WHEN** правило `who_free_sugar` нарушено за день и один из продуктов приёма имеет вклад в сахар >30% от дневного значения
- **THEN** карточка приёма показывает строку «Свободный сахар — 45% дневного» с красным индикатором

#### Scenario: Приём с несколькими нарушениями
- **WHEN** в приёме есть продукты с вкладом >30% в два разных нарушенных правила
- **THEN** карточка показывает оба правила списком, каждое с названием и процентом вклада

#### Scenario: Приём без нарушений
- **WHEN** ни один продукт в приёме не имеет вклада >30% в нарушенные правила
- **THEN** карточка показывает «Все правила соблюдены» зелёным цветом

#### Scenario: Приём без данных (skipped / empty)
- **WHEN** приём имеет статус `skipped` или `empty`
- **THEN** секция нарушений не отображается

### Requirement: Визуальное оформление подсветки
Система SHALL использовать единую цветовую схему: красный для violation, жёлтый для warning, нейтральный для normal.

#### Scenario: Нарушение зоны violation
- **WHEN** правило нарушено в зоне `violation`
- **THEN** индикатор правила в карточке приёма красный, текст красный

#### Scenario: Нарушение зоны warning
- **WHEN** правило нарушено в зоне `warning`
- **THEN** индикатор правила в карточке приёма жёлтый, текст жёлтый

## ADDED Requirements

### Requirement: Подсветка продукта по вкладу в нарушение
Система SHALL подсвечивать продукт в карточке приёма цветным кружком и стрелкой раскрытия, если правило нарушено на уровне дня и вклад этого продукта в метрику правила превышает 30%. При тапе продукт раскрывается через `DisclosureGroup`, показывая список нарушенных правил с процентами вклада.

#### Scenario: Продукт с большим вкладом в сахар
- **WHEN** правило `who_free_sugar` нарушено (сахар >10% калорий за день) и конкретный ингредиент даёт 45% дневного сахара
- **THEN** продукт получает красный кружок, при раскрытии показывает «Свободный сахар — 45% дневного»

#### Scenario: Продукт с недостаточным вкладом
- **WHEN** правило `who_free_sugar` нарушено, но ингредиент даёт только 15% дневного сахара
- **THEN** продукт не подсвечивается

#### Scenario: Правило не нарушено
- **WHEN** правило `who_sodium` находится в зоне `normal` за день
- **THEN** ни один продукт не подсвечивается по этому правилу, даже если содержание натрия высокое

#### Scenario: Один продукт нарушает несколько правил
- **WHEN** продукт «Сливочное масло» даёт 52% насыщенных жиров и 38% общих жиров, оба правила нарушены
- **THEN** при раскрытии показываются оба правила с соответствующими процентами
