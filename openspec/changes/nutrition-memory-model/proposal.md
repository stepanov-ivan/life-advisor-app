## Why

Текущему агентскому потоку не хватает устойчивой и объяснимой памяти питания: рекомендации и уточнения часто не персонализированы, а пользователю приходится повторно вводить похожие данные. Нужна минимальная модель памяти, которая снижает трение ввода, повышает релевантность советов и сохраняет прозрачность решений агента.

## What Changes

- Ввести минимальную объяснимую память питания, которая хранит:
  - структурированные сигналы по приёмам пищи,
  - гипотезы предпочтений/ограничений,
  - пробелы данных (`data-gap`) для последующих уточнений.
- Зафиксировать, что **структура** (ккал/БЖУ/граммы) является источником истины, а текст — слоем быстрого ввода.
- Добавить локальные подсказки ввода из последних 50 структурированных логов (без LLM на каждый символ), с матчингом префикс + похожесть.
- При выборе подсказки заполнять структуру сразу и маркировать источник как `memory_suggestion`; LLM вызывать только при `out_of_sync` или явном действии «Переоценить».
- Если после выбора подсказки текст изменён — помечать состояние `out_of_sync` и по умолчанию предлагать «Переоценить по тексту».
- При отсутствии веса/объёма не блокировать поток: сохранять результат с `low confidence` и `data-gap`, а также показывать быстрые варианты уточнения порции.
- Хранить косвенные выводы как гипотезы (не как факты), подтверждать их только после повторяющихся сигналов; подтверждённое хранить как мягкое предпочтение с редкой перепроверкой при конфликте.
- Обновлять шаблон памяти после ручной правки структуры только если изменение калорий не превышает 30% от исходной структуры.
- Добавить пользовательский контроль памяти: точечное удаление элементов памяти и полный сброс.

## Capabilities

### New Capabilities
- `nutrition-memory`: Минимальная, пользовательская и объяснимая модель памяти питания (структура логов, гипотезы, data-gap, применение сигналов агентом).
- `meal-input-suggestions`: Локальные подсказки по истории структурированных приёмов пищи с быстрым заполнением структуры.

### Modified Capabilities
- `llm-agent`: Изменение поведения агента при использовании памяти (мягкое применение, `out_of_sync`, выборочные уточнения, прозрачная маркировка источника).
- `local-storage`: Добавление/уточнение сущностей и полей для памяти, гипотез, data-gap, источников структуры и управления жизненным циклом памяти.
- `event-logging`: Изменение потока логирования — разведение свободного текста и структурного источника истины, поддержка сценария «подсказка → структура».
- `insight-engine`: Использование сигналов памяти и confidence/data-gap маркеров при формировании рекомендаций.

## Impact

- Затронутые UI-потоки:
  - ввод приёма пищи (подсказки, быстрый выбор порции, состояния `out_of_sync`),
  - редактирование структуры,
  - управление памятью (точечное удаление/сброс).
- Затронутые данные:
  - структурные записи приёмов,
  - источники структуры (`memory_suggestion` / `llm`),
  - гипотезы предпочтений,
  - data-gap и confidence-сигналы.
- Затронутые алгоритмы:
  - локальный ранжир подсказок (префикс + похожесть),
  - правила подтверждения гипотез,
  - правила обновления памяти после ручных правок.
- Риски и ограничения:
  - риск накопления шумных шаблонов (смягчается порогом 30%),
  - риск устаревания гипотез (смягчается редкой перепроверкой при конфликте),
  - необходимость прозрачной маркировки источника структуры для отладки и доверия пользователя.
