## ADDED Requirements

### Requirement: Объяснимая память питания
Система SHALL хранить минимальную объяснимую память питания, включающую структурные сигналы приёмов пищи, гипотезы предпочтений и записи пробелов данных (`data-gap`).

#### Scenario: Сохранение сигнала питания
- **WHEN** пользователь сохраняет структурированный приём пищи
- **THEN** система сохраняет структурный сигнал в памяти с привязкой ко времени и источнику структуры

### Requirement: Гипотезы предпочтений
Система SHALL формировать гипотезу предпочтения/ограничения только после повторяющихся сигналов и не превращать её в факт без явного пользовательского подтверждения.

#### Scenario: Создание гипотезы после повторов
- **WHEN** обнаружены 2 повторяющихся сигнала по одной теме (например, исключение рыбы)
- **THEN** система создаёт гипотезу со статусом `pending_confirmation`

#### Scenario: Подтверждение гипотезы
- **WHEN** пользователь выбирает `Да` в inline-подтверждении
- **THEN** гипотеза помечается как мягкое предпочтение и участвует в рекомендациях

### Requirement: Отложенное подтверждение
Система SHALL поддерживать отложенное подтверждение гипотезы с повторным запросом не ранее чем через 3 дня.

#### Scenario: Спросить позже
- **WHEN** пользователь выбирает `Спросить позже`
- **THEN** повторный запрос по этой гипотезе не показывается в течение 3 дней

### Requirement: Контроль памяти пользователем
Система SHALL предоставлять точечное удаление элемента памяти и полный сброс памяти.

#### Scenario: Точечное удаление
- **WHEN** пользователь удаляет конкретную запись памяти
- **THEN** запись удаляется и перестаёт использоваться в подсказках и рекомендациях

#### Scenario: Полный сброс
- **WHEN** пользователь запускает полный сброс памяти
- **THEN** все memory-записи удаляются
