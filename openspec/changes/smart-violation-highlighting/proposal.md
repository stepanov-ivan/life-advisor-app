## Why

Текущая система подсветки нарушений создаёт информационный шум: статический порог 30% заставляет подсвечивать каждый продукт во множестве правил, к которым он имеет косвенное отношение. Например, кефир подсвечивается за «отсутствие клетчатки», «отсутствие овощей», «влияние на углеводы», «свободный сахар» и «белок» — пользователь видит 5+ нарушений на одном продукте, большинство из которых тривиальны и нерелевантны. Экран правил перегружен индикаторами зон, хотя его основная функция — настройка (вкл/выкл). Текстовки на карточке приёма пищи не содержат названий правил, что делает их непонятными. Нужна настройка доменной логики фильтрации, чтобы показывать только значимые, релевантные нарушения.

## What Changes

- **Убрать индикаторы зон с экрана правил** — экран `RulesListView` становится чисто настроечным: список правил по категориям с описаниями и toggle вкл/выкл, без цветовых кружков, зоновых текстов и сводной статистики
- **Исключить `category_missing` из подсветки продуктов** — продукт не должен подсвечиваться за то, что он «не цельнозерновой», «не овощ» и т.д. Это нарушение дня, а не конкретного продукта
- **Исключить `below_lower` / `approaching_lower` из подсветки продуктов** — продукт не виноват, что в нём мало белка, клетчатки или другого нутриента. Нарушение остаётся на уровне дня
- **Поднять порог вклада продукта с 30% до 33%** — для range-правил с превышением (exceeds_upper, approaching_upper) продукт подсвечивается, только если его вклад >33% дневного значения
- **Исправить текстовки нарушений** — формат «Название правила: значение при норме порог» вместо «Превышение: значение при норме порог»
- **Оставить `unwanted_category_present` без изменений** — продукт с нежелательной категорией (ультра-обработанное, обработанное мясо) подсвечивается как причина нарушения

## Capabilities

### New Capabilities
- `smart-violation-filtering`: Набор правил доменной логики для фильтрации релевантности подсветки продукта:
  - `category_missing` → продукт не подсвечивается
  - `below_lower` / `approaching_lower` → продукт не подсвечивается
  - `exceeds_upper` / `approaching_upper` → топ-3 продукта с вкладом >33%
  - `unwanted_category_present` → продукт подсвечивается (без изменений)

### Modified Capabilities
- `nutrition-rules`: Удаление требования «Daily freshness статуса правила» — экран правил больше не показывает зоновые индикаторы, зоновые тексты и сводную статистику; остаются категории-секции, описания правил и toggle вкл/выкл
- `meal-rule-feedback`: Изменение формата текстовок нарушений — добавление названия правила: «Название правила: значение при норме порог»; интеграция новых правил фильтрации релевантности
- `product-violation-highlighting`: Замена статического порога 30% на порог 33%; исключение `category_missing` и `below_lower`/`approaching_lower` из подсветки продуктов

## Impact

- `RuleEngine.swift` — изменение `contributionForRangeRule` (поднять порог 30%→33%, исключить below_lower/approaching_lower), изменение `contributionForPresenceRule` (исключить category_missing), изменение `violationDescription(for:)` (добавить название правила)
- `RulesListView.swift` — удаление `zoneIndicator`, `zoneLabel`, `summaryCircle`, `summary`, сводной строки
- `MealSlotCard.swift` — обновление логики `productViolationMap` под новые правила фильтрации
- `DayAnalyticsPanel.swift` — без изменений
- `RuleDetailView.swift` — без изменений (violationDescription обновится автоматически)
- `RuleViolation` модель — без изменений (reasonCode сохраняется)
