## ADDED Requirements

### Requirement: Исключение category_missing из подсветки продуктов
Система SHALL NOT подсвечивать продукты для presence-правил с `reasonCode = "category_missing"`. Нарушение отсутствия категории (цельные злаки, овощи, фрукты, бобовые) относится ко дню в целом, а не к конкретному продукту. `contributionJSON` для таких нарушений SHALL быть `nil`.

#### Scenario: Кефир не подсвечивается за отсутствие цельных злаков
- **WHEN** правило `whole_grain` нарушено (`reasonCode = "category_missing"`)
- **THEN** ни один продукт не получает цветной индикатор по этому правилу, нарушение не показывается в карточках приёмов

#### Scenario: Отсутствие овощей — продукты не виноваты
- **WHEN** правило `fruit_veg` нарушено (`reasonCode = "category_missing"`)
- **THEN** ни один продукт не подсвечивается, нарушение видно только в `DayAnalyticsPanel`

### Requirement: Исключение below_lower и approaching_lower из карточек приёма
Система SHALL NOT показывать нарушения с `reasonCode = "below_lower"` или `"approaching_lower"` в карточках приёма (`MealSlotCard`). Продукт не виноват в недостатке нутриента — нарушение относится к дню в целом и отображается только в `DayAnalyticsPanel`.

#### Scenario: Белок ниже нормы — не показывается в карточке
- **WHEN** правило `protein` нарушено (`reasonCode = "below_lower"`), белок = 8% при норме ≥10%
- **THEN** нарушение отсутствует в `MealSlotCard`, видно только в `DayAnalyticsPanel`

#### Scenario: Клетчатка ниже нормы — не показывается в карточке
- **WHEN** правило `fiber` нарушено (`reasonCode = "below_lower"`), клетчатка = 18 г при норме ≥25 г
- **THEN** нарушение отсутствует в `MealSlotCard`

### Requirement: Подсветка продуктов только при превышении верхней границы
Система SHALL подсвечивать продукты в `MealSlotCard` для range-правил только при `reasonCode = "exceeds_upper"` или `"approaching_upper"`. Топ-3 продукта по вкладу в нутриент с вкладом >33% получают цветной индикатор и стрелку раскрытия.

#### Scenario: Сахар превышен — продукт с вкладом >33% подсвечивается
- **WHEN** правило `added_sugar` нарушено (`reasonCode = "exceeds_upper"`), пирожное даёт 45% дневного сахара
- **THEN** пирожное получает красный индикатор и раскрывается с процентом вклада

#### Scenario: Сахар превышен, но вклад продукта менее 33%
- **WHEN** правило `added_sugar` нарушено, 10 продуктов имеют вклад в сахар, но ни один не превышает 33%
- **THEN** ни один продукт не подсвечивается — нарушение показывается только текстом в карточке

#### Scenario: Насыщенные жиры превышены — масло в топ-3
- **WHEN** правило `saturated_fat` нарушено (`reasonCode = "exceeds_upper"`), масло даёт 52% дневного насыщенного жира
- **THEN** масло получает красный индикатор и раскрывается с процентом вклада

### Requirement: Подсветка продукта для unwanted_category_present
Система SHALL подсвечивать продукт для presence-правил с `reasonCode = "unwanted_category_present"`. Продукт, содержащий нежелательную категорию, является прямой причиной нарушения.

#### Scenario: Ультра-обработанный продукт подсвечивается
- **WHEN** правило `ultra_processed` нарушено (`reasonCode = "unwanted_category_present"`), продукт «Чипсы» имеет категорию `ultra_processed`
- **THEN** «Чипсы» получает красный индикатор

#### Scenario: Обработанное мясо подсвечивается
- **WHEN** правило `processed_meat` нарушено (`reasonCode = "unwanted_category_present"`), продукт «Колбаса» имеет категорию `processed_meat`
- **THEN** «Колбаса» получает красный индикатор

### Requirement: Ограничение количества подсвеченных продуктов
Система SHALL ограничивать подсветку продуктов тремя наибольшими по вкладу для каждого нарушенного правила с превышением.

#### Scenario: Более трёх продуктов с вкладом >33%
- **WHEN** 5 продуктов имеют вклад >33% в нарушенное правило `sodium`
- **THEN** подсвечиваются только топ-3 по вкладу

#### Scenario: Менее трёх продуктов с вкладом >33%
- **WHEN** только 1 продукт имеет вклад >33%
- **THEN** подсвечивается только этот продукт
