## MODIFIED Requirements

### Requirement: Редактирование приёма пищи
Пользователь SHALL иметь возможность редактировать структуру независимо от текста; изменение граммовки MUST автоматически пересчитывать Ккал/Б/Ж/У без отдельного действия сброса.

#### Scenario: Правка структуры без изменения текста
- **WHEN** пользователь редактирует только структуру
- **THEN** текст может оставаться прежним, а источником истины остаётся структура

#### Scenario: Автопересчёт при изменении граммовки
- **WHEN** пользователь изменил значение `г` у позиции
- **THEN** система автоматически пересчитывает Ккал/Б/Ж/У для этой позиции от baseline-значений без отдельной кнопки

#### Scenario: Перезапись ручных макросов изменением граммовки
- **WHEN** пользователь ранее вручную менял Ккал/Б/Ж/У позиции, а затем изменил `г`
- **THEN** система снимает lock ручного режима и перезаписывает макросы автопересчётом

#### Scenario: Таблица состава без единиц измерения
- **WHEN** пользователь открывает редактор структурированного приёма пищи
- **THEN** секция `Состав блюда` показывает колонки `г | Ккал | Б | Ж | У` без колонки `ед`

#### Scenario: Формат дробных значений
- **WHEN** пользователь вводит дробные значения граммовки
- **THEN** система принимает дробные значения и отображает `Ккал/Б/Ж/У` с точностью до 1 знака после запятой

### Requirement: Синхронизация текста и структуры
Система SHALL хранить и обновлять состояние синхронизации текста и структуры на уровне события (`structureOutdated`, `structureLastSyncedAt`) без блокировки сохранения.

#### Scenario: Смысловое изменение текста помечает рассинхрон
- **WHEN** normalized-текст (trim + схлопывание пробелов) отличается от `baselineTextNormalized`
- **THEN** система устанавливает `structureOutdated=true`

#### Scenario: Изменение только пробелов
- **WHEN** пользователь изменил только форматирование пробелов, не меняя normalized-текст
- **THEN** система не меняет состояние синхронизации структуры

#### Scenario: Сохранение при `structureOutdated=true`
- **WHEN** пользователь сохраняет событие с неактуальной структурой
- **THEN** система сохраняет текущие `items` как канон, сохраняет текст пользователя и персистит флаг `structureOutdated`

#### Scenario: Ручная правка структуры не снимает рассинхрон
- **WHEN** пользователь вручную изменил `items` без успешного LLM-пересчёта
- **THEN** система сохраняет `structureOutdated=true`

#### Scenario: Инициализация синхронизации для нового события
- **WHEN** впервые получена валидная структура для нового события (из памяти или LLM)
- **THEN** система сохраняет `baselineTextNormalized`, устанавливает `structureOutdated=false` и обновляет `structureLastSyncedAt`
