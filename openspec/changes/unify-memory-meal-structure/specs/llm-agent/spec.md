## MODIFIED Requirements

### Requirement: Структуризация текста через LLM
Система SHALL отправлять текст логирования в LLM и получать строгий JSON-контракт оценки питания, где каждая позиция содержит чистое имя и порцию в граммах.

#### Scenario: Строгий контракт позиции
- **WHEN** LLM возвращает структурированный ответ
- **THEN** каждый item содержит обязательные поля `name` и `grams > 0`, без `unit`

#### Scenario: Чистое имя позиции
- **WHEN** LLM формирует item
- **THEN** поле `name` не содержит порционных маркеров, скобок и служебных обозначений (например, `~порция`, `3 шт`, `150г`)

#### Scenario: Невалидный контракт
- **WHEN** хотя бы один item нарушает контракт (`grams` отсутствует/невалиден или `name` грязный)
- **THEN** система помечает оценку как `parseFailed` целиком и не применяет частичный результат

### Requirement: Fallback реконструкции структуры
Система SHALL выполнять LLM-реконструкцию структуры как обязательный fallback при неполных memory-items и не сохранять событие при ошибке fallback.

#### Scenario: Fallback при неполных данных памяти
- **WHEN** memory-path не даёт валидный канонический набор позиций
- **THEN** система отправляет текст в LLM для полной реконструкции состава до сохранения события

#### Scenario: Ошибка fallback
- **WHEN** fallback-вызов LLM завершился ошибкой сети/парсинга
- **THEN** система не сохраняет событие и возвращает ошибку пользователю

### Requirement: Явный пересчёт по действию пользователя
Система SHALL пересчитывать структуру по отредактированному тексту только по явному действию пользователя.

#### Scenario: Редактирование текста без кнопки пересчёта
- **WHEN** пользователь изменил текст и не инициировал явный пересчёт
- **THEN** система не выполняет автоматический LLM-вызов и сохраняет текущую структуру

#### Scenario: Явный пересчёт без фактических изменений
- **WHEN** пользователь запрашивает пересчёт, но normalized-текст совпадает с baseline
- **THEN** система пропускает LLM-вызов как no-op и не обновляет `structureLastSyncedAt`

#### Scenario: Успешный явный пересчёт
- **WHEN** пользователь запрашивает пересчёт и LLM возвращает валидный результат
- **THEN** система обновляет канонический `MealEvent.items`, сбрасывает `structureOutdated` и обновляет `structureLastSyncedAt`

#### Scenario: Ошибка явного пересчёта
- **WHEN** пользователь запрашивает пересчёт и LLM возвращает ошибку
- **THEN** система сохраняет текущие `items` без изменений, оставляет `structureOutdated=true` и позволяет сохранить событие без пересчёта
