## MODIFIED Requirements

### Requirement: Текстовый ввод приёма пищи
Пользователь SHALL иметь возможность логировать приём пищи свободным текстом с опциональным выбором подсказки, где структура выступает источником истины.

#### Scenario: Свободный ввод без подсказки
- **WHEN** пользователь вводит текст вручную и сохраняет лог
- **THEN** система формирует структуру через LLM и сохраняет её как каноническую

#### Scenario: Ввод с подсказкой
- **WHEN** пользователь выбирает подсказку
- **THEN** структура заполняется сразу, а лог сохраняется без автоматической отправки до явного подтверждения

### Requirement: Редактирование приёма пищи
Пользователь SHALL иметь возможность редактировать структуру независимо от текста; при рассинхроне система MUST предложить переоценку по тексту по умолчанию.

#### Scenario: Правка структуры без изменения текста
- **WHEN** пользователь редактирует только структуру
- **THEN** текст может оставаться прежним, а источником истины остаётся структура

#### Scenario: Рассинхрон текста и структуры
- **WHEN** пользователь изменил текст после подстановки структуры
- **THEN** система помечает структуру как потенциально неактуальную и предлагает действие «Пересчитать по тексту»

#### Scenario: Пользователь оставляет структуру при рассинхроне
- **WHEN** структура помечена как потенциально неактуальная и пользователь выбирает действие «Оставить структуру»
- **THEN** система сохраняет структуру как источник истины, а текст сохраняет только как заметку без влияния на расчёт

#### Scenario: Ошибка пересчёта по тексту
- **WHEN** пользователь запускает «Пересчитать по тексту» и LLM возвращает ошибку
- **THEN** система сохраняет текущую структуру без изменений и показывает сообщение об ошибке

## ADDED Requirements

### Requirement: Упрощённая карточка приёма пищи на дашборде
Система SHALL отображать на основной карточке только ключевые пользовательские данные приёма пищи без технических меток диагностики.

#### Scenario: Карточка структурированного приёма пищи
- **WHEN** пользователь просматривает карточку приёма пищи на дашборде
- **THEN** система не показывает технические метки `Уверенность` и `Из памяти`

### Requirement: Состав блюда как основная структура
Система SHALL отображать и редактировать структуру блюда в секции `Состав блюда` с табличными столбцами `кол-во | ед | Ккал | Б | Ж | У`.

#### Scenario: Отображение заголовков структуры
- **WHEN** пользователь открывает редактор структурированного приёма пищи
- **THEN** система показывает секцию `Состав блюда` с заголовками `кол-во | ед | Ккал | Б | Ж | У`

#### Scenario: Поддержка количества и единиц
- **WHEN** пользователь редактирует позицию структуры
- **THEN** система позволяет вводить дробное количество и выбирать единицу из `g`, `ml`, `pcs`, где значение по умолчанию — `g`

#### Scenario: Пустая структура после удаления позиций
- **WHEN** пользователь удалил все позиции структуры
- **THEN** система запрашивает подтверждение и переводит событие в статус ожидания пересчёта
