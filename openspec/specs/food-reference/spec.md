## REMOVED Requirements

### Requirement: Seed-заполнение справочника
**Reason**: В agent-first подходе локальный справочник продуктов полностью исключается из приложения и больше не является частью функционального контракта.
**Migration**: Удалить seed-механику и связанные данные `FoodItem`; оценка выполняется через LLM по free-text.

### Requirement: Поиск продукта по названию
**Reason**: Поиск по справочнику больше не нужен после полного удаления справочника из UX и бизнес-логики.
**Migration**: Удалить UI и сервисы поиска `FoodItem`; ввод пищи выполняется только через текст и агентскую оценку.

### Requirement: Добавление пользовательского продукта
**Reason**: Ручное управление справочником убрано из продукта в пользу агентского потока и памяти правок.
**Migration**: Отключить создание `FoodItem` пользователем; персонализация переносится в memory-слой корректировок.

### Requirement: Расчёт БЖУ для ингредиента
**Reason**: Расчёт БЖУ через `FoodItem` не используется в основном продукте и удаляется вместе со справочником.
**Migration**: Итоговые макросы и компонентные оценки формируются из LLM-контракта.

### Requirement: Поиск продукта по частичному совпадению
**Reason**: Частичный поиск по справочнику теряет смысл при полном удалении справочника.
**Migration**: Удалить соответствующие поля/экраны поиска; оставить только текстовый логинг.

### Requirement: Группировка по категориям
**Reason**: Категорийный UI справочника удаляется как часть полного отказа от food-reference capability.
**Migration**: Удалить категории и список справочника из интерфейса и модели данных.
