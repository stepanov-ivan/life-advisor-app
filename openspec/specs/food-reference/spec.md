## Requirements
### Requirement: Seed-заполнение справочника
Система SHALL при первом запуске загружать продукты из JSON-файла в бандле приложения в таблицу `FoodItem` SwiftData. Каждый продукт содержит поля: `name`, `category`, `calories`, `proteins`, `fats`, `carbs`, `fiber` (на 100г).

#### Scenario: Первый запуск — заполнение справочника
- **WHEN** приложение запускается впервые и `FoodItem`-таблица пуста
- **THEN** парсится JSON-файл из бандла, создаются записи `FoodItem` для каждого продукта (500–1000 позиций) с категориями: крупы, мясо, овощи, фрукты, молочка, напитки, полуфабрикаты

#### Scenario: Повторный запуск — пропуск seed
- **WHEN** приложение запускается повторно и таблица `FoodItem` уже содержит данные
- **THEN** seed-заполнение не выполняется, пользовательские продукты сохранены

### Requirement: Поиск продукта по названию
Система SHALL обеспечивать поиск продуктов в справочнике по названию с поддержкой точного и неточного совпадения.

#### Scenario: Точное совпадение
- **WHEN** выполняется поиск `FoodItem` по названию «овсянка»
- **THEN** возвращается запись с `name == "овсянка"`

#### Scenario: Нечёткое совпадение
- **WHEN** LLM возвращает название «овсяные хлопья», которого нет в справочнике
- **THEN** система выполняет fuzzy-поиск и находит ближайшее совпадение «овсянка»

### Requirement: Добавление пользовательского продукта
Пользователь SHALL иметь возможность добавить собственный продукт в справочник с указанием названия и БЖУ на 100г.

#### Scenario: Создание пользовательского продукта
- **WHEN** пользователь вводит название «протеиновый батончик» и БЖУ (350 ккал, 25г белка, 10г жиров, 40г углеводов на 100г)
- **THEN** создаётся `FoodItem`, доступный для поиска наравне со стандартными

### Requirement: Расчёт БЖУ для ингредиента
Система SHALL вычислять БЖУ для ингредиента на основе данных из `FoodItem` и указанного количества.

#### Scenario: Расчёт для 100г овсянки
- **WHEN** `FoodItem` «овсянка» имеет 342 ккал на 100г, и LLM вернула `amount: 100, unit: "г"`
- **THEN** `Ingredient` получает `calories = 342, proteins = 12, fats = 6, carbs = 60` (пропорционально количеству)

#### Scenario: Расчёт для нестандартного количества
- **WHEN** `FoodItem` «молоко» имеет 42 ккал на 100г, и LLM вернула `amount: 200, unit: "мл"`
- **THEN** `Ingredient` получает `calories = 84` (пропорционально 200 мл ≈ 200 г)

### Requirement: Поиск продукта по частичному совпадению
Пользователь SHALL иметь возможность искать продукты в справочнике по части названия при структурированном вводе.

#### Scenario: Частичный поиск
- **WHEN** пользователь вводит «ябл» в поле поиска справочника
- **THEN** возвращаются все продукты, содержащие «ябл» в названии (яблоко, яблочный сок, яблочное пюре)

### Requirement: Группировка по категориям
Система SHALL отображать продукты в справочнике сгруппированными по категориям.

#### Scenario: Выбор категории
- **WHEN** пользователь открывает справочник продуктов
- **THEN** продукты сгруппированы по категориям (крупы, мясо, овощи, фрукты, молочка, напитки, полуфабрикаты), каждая категория раскрывается
