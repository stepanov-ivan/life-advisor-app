## MODIFIED Requirements

### Requirement: Структуризация текста через LLM
Система SHALL отправлять текст логирования в LLM и получать строгий JSON-контракт оценки питания, а также использовать память мягко: без блокировки ввода, с прозрачной маркировкой источника и повторной оценкой только при необходимости.

#### Scenario: Оценка после free-text ввода
- **WHEN** пользователь вводит новый свободный текст без готовой структуры
- **THEN** система отправляет текст в LLM и сохраняет результат как источник `llm`

#### Scenario: Структура из памяти без переоценки
- **WHEN** пользователь выбрал подсказку и структура валидна
- **THEN** система сохраняет лог без LLM-вызова и помечает источник как `memory_suggestion`

#### Scenario: Переоценка при рассинхроне
- **WHEN** после выбора подсказки пользователь изменил текст и состояние стало `out_of_sync`
- **THEN** при сохранении по умолчанию запускается переоценка по тексту через LLM

## ADDED Requirements

### Requirement: Неблокирующая обработка data-gap
Система SHALL при отсутствии веса/объёма сохранять оценку с `low confidence` и `data-gap`, не блокируя поток логирования.

#### Scenario: Нет веса порции
- **WHEN** текст не содержит достаточных данных о порции
- **THEN** лог сохраняется с `low confidence`, а `data-gap` добавляется для последующего уточнения
