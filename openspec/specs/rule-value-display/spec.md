## Purpose

Отображение расчётных значений метрик правил вместе с порогами в аналитической панели дашборда вместо одних только цветовых индикаторов.

## Requirements

### Requirement: Отображение числового значения метрики в панели
Система SHALL показывать в `DayAnalyticsPanel` справа от названия правила фактическое значение метрики и порог вместо зонового текста («В норме» / «Нарушено»). Значение SHALL быть окрашено в цвет зоны правила.

#### Scenario: Правило с двумя границами в норме
- **WHEN** правило `who_protein` оценивается за день, значение белка = 12,3% калорий, зона = normal
- **THEN** строка показывает «12,3% из 10–15%» зелёным цветом

#### Scenario: Правило с верхней границей нарушено
- **WHEN** правило `who_free_sugar` оценивается, значение сахара = 14,2% калорий, порог ≤10%, зона = violation
- **THEN** строка показывает «14,2% из ≤10%» красным цветом

#### Scenario: Правило с нижней границей в warning
- **WHEN** правило `who_fiber` оценивается, значение клетчатки = 22 г, порог ≥25 г, зона = warning
- **THEN** строка показывает «22 г из ≥25 г» жёлтым цветом

#### Scenario: Правило без данных
- **WHEN** за день нет structured-приёмов, правило `who_protein` оценивается
- **THEN** строка показывает «Нет данных» серым цветом

### Requirement: Маппинг field → единица измерения
Система SHALL определять единицу измерения и формат числа по полю `field` правила.

#### Scenario: Процентное поле
- **WHEN** правило имеет field `proteinPercent`, `fatPercent`, `carbsPercent`, `saturatedFatPercent`, `transFatPercent`, `sugarPercent`, `energyBalancePercent` или `pufaPercent`
- **THEN** значение умножается на 100, округляется до 1 знака, единица «%»

#### Scenario: Поле в миллиграммах
- **WHEN** правило имеет field `sodiumMg`
- **THEN** значение округляется до 0 знаков, единица « мг»

#### Scenario: Поле в граммах
- **WHEN** правило имеет field `fiberGrams`, `fruitVegGrams` или `redMeatGrams`
- **THEN** значение округляется до 0 знаков, единица « г»

### Requirement: Формат пороговых значений
Система SHALL форматировать порог правила в зависимости от наличия `lower` и `upper`:

#### Scenario: Обе границы
- **WHEN** правило имеет `lower: 0.10` и `upper: 0.15`
- **THEN** порог форматируется как «10–15%»

#### Scenario: Только верхняя граница
- **WHEN** правило имеет только `upper: 0.10`
- **THEN** порог форматируется как «≤10%»

#### Scenario: Только нижняя граница
- **WHEN** правило имеет только `lower: 25`
- **THEN** порог форматируется как «≥25 г»

### Requirement: Формат presence и countSkipped правил
Система SHALL показывать presence-правила с зоновым текстом, а countSkipped — с количеством пропусков.

#### Scenario: Presence-правило выполнено
- **WHEN** правило `who_whole_grain` в зоне normal
- **THEN** строка показывает «В норме» зелёным

#### Scenario: Presence-правило нарушено
- **WHEN** правило `who_processed_meat` в зоне violation
- **THEN** строка показывает «Нарушено» красным

#### Scenario: CountSkipped с пропусками
- **WHEN** правило `who_meal_regularity` показывает 3 пропуска за неделю
- **THEN** строка показывает «3 пропуска из ≤3»
